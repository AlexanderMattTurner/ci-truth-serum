- New Tier 2 hook `check-requires-concurrency`: requires every `pull_request`/`pull_request_target`-triggered workflow to declare a `concurrency:` block, so a new push cancels the superseded run instead of stacking a second full run on a capped runner pool. This closes the gap left by `check-concurrency`, which only validates a block that already exists. The block may sit at the workflow level or on any job (the doctrine-approved shape for a required check that needs serialization). Push-only, schedule-only, and reusable (`workflow_call`) workflows are exempt. Opt out with `# concurrency-not-required` when queue-don't-cancel is intended.
