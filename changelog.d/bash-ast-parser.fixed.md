- `check-workflow-pipefail` and `check-flag-arity` now parse shell with a real bash grammar (`tree-sitter-bash`) instead of hand-rolled quote/heredoc/case tokenizers, closing the blind spots those approximations had. `check-workflow-pipefail` now sees a masking pipe that an escaped quote (`echo "a\"" | tee`), a `$'…'` ANSI-C string, or nested `$()`/backtick command substitution previously hid from its char-by-char quote counter. `check-flag-arity` no longer loses a violation when a `;;` or `case … in` appears inside a string, and it excludes reads in nested `case` arms by construction. Both lints gain `tree-sitter` / `tree-sitter-bash` as runtime dependencies (declared in each hook's `additional_dependencies`).
