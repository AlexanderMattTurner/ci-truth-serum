- New Tier 1 honesty lint `check-pipefail-grep-pipe` bans `producer | grep -q` under `set -o pipefail`. When `grep -q` matches early it closes the pipe and the still-writing producer dies with `SIGPIPE` (exit 141); `pipefail` then surfaces 141 as the pipeline status, so a genuine match is misread as no-match once the producer's output outgrows the ~64 KiB pipe buffer. The lint flags a pipefail-scoped shell file (including sourced `# shellcheck shell=bash` libraries, which inherit strict mode from their callers) that pipes a streaming producer into a quiet grep, and directs authors to the capture-then-here-string fix (`out="$(producer)"; grep -q PAT <<<"$out"`). Bounded builtin producers (`echo`/`printf`/`:`) are exempt; a provably-tiny non-builtin case opts out with `# pipefail-grep-ok: <reason>`.
