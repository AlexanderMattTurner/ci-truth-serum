- `check-workflow-pipefail`, `check-inline-run-length`, and `check-externalized-markers` now report a workflow file they can't parse as YAML as a violation, instead of silently treating it as clean. A malformed workflow used to read as "no findings" — a false-green on the exact file the lint exists to verify; YAML syntax is still actionlint's job, but a document PyYAML can't even build is no longer waved through.
