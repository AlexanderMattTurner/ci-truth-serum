- `check-secret-file-perms` (Extras, picked up automatically by the `check-extras` aggregate): flags a shell line that creates a secret-named file (path matching `token`/`secret`/`cred`/`key`/`passwd`/`password`/`npmrc`/`auth`/`pem`/`refresh`/`cookie`/`id_rsa`) world-readable — via a `>`/`>>` redirect, `touch`, `tee`, or non-private `install` — and only `chmod 0?[46]00`s it private within the next ~3 non-blank lines. That later chmod is the tell that the author knew the file must be private but created it readable first, leaving a window a co-tenant can read. Guarded creates never fire: a standing `umask 0?77` earlier in the file, an inline `(umask 077; …)` on the create line, or `install -m 0?[46]00`. A create with no nearby chmod is not flagged (near-zero false positives). Suppress a deliberate case with `# secret-perms-ok: <reason>` on the create line.
