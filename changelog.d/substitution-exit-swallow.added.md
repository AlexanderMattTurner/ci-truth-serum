- `check-substitution-exit-swallow` (Tier 1 · Honesty): flags a structured-data producer (`jq`/`yq`) feeding a shell loop through a construct that discards its exit status — `done < <(jq …)` / `mapfile < <(yq …)` (process-substitution stdin redirect) and `jq … | while read` (pipeline). Both report on the redirect/pipe, not on the producer, so a query or data error makes the loop iterate zero times while the function returns success — a fail-open. Producer set is curated to `jq`/`yq` (whose non-zero exit is a genuine fail-closed signal); `grep`/`cat`/`find`/`sed` are excluded. Opt out with a reason-bearing `# allow-substitution-exit: <reason>`. Included in the `check-tier1` aggregate.
