- The shell-analyzing lints (`check-case-default`, `check-flag-arity`, `check-workflow-pipefail`, `check-pinned-downloads`) no longer crash the whole pre-commit/CI run on adversarial input: tree-sitter-bash 0.25.1 has a native heap-corruption bug that segfaults its parser on certain byte mixes (high-plane codepoints, stacked combining marks, zero-width / BOM / paragraph-separator, C0 controls). The shared parser now maps every non-ASCII/control byte to `x` before parsing — length-preserving, so node offsets and line numbers are unchanged and no structural finding differs (shell syntax is ASCII; non-ASCII only ever lives in opaque string/comment/heredoc bodies) — turning the crash into a normal parse.
