- `check-trusted-base` (Tier 1, Security): flags the pwn-request shape — a `pull_request(_target)` job that checks out the PR **head** ref _and_ runs privileged (write `permissions:` or a `secrets.*` in `env:`), so the PR author's code executes with the base repo's token and secrets. Read-only, secret-free head checkouts are not flagged; opt out with `# trusted-base-ok: <reason>`.
