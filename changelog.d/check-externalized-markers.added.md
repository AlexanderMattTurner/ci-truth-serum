- `check-externalized-markers` (Tier 2, opt-in): flags any workflow job where a policy marker (default: the git history-rewrite commands `git commit --amend` / `git rebase` / `git filter-branch` / force-pushes) is reachable only through `bash .github/scripts/*.sh` or `uses: ./.github/actions/*` indirection, never in an inline `run:`. That delta is the exact blind spot of a guard that scans only inline `run:` text — such a guard passes vacuously once the command is externalized. Add repo-specific markers with `--marker '<cmd>'`.
