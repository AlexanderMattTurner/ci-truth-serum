- `check-externalized-markers` (Tier 2, opt-in): flags any workflow job where a policy marker (default: the git history-rewrite commands `git commit --amend` / `git rebase` / `git filter-branch` / force-pushes) is reachable only through `bash .github/scripts/*.sh` or `uses: ./.github/actions/*` indirection, never in an inline `run:`. That delta is the exact blind spot of a guard that scans only inline `run:` text — such a guard passes vacuously once the command is externalized. Markers match whole command tokens (so `git push --force` doesn't fire inside `git push --force-with-lease`). Add repo-specific markers with `--marker '<cmd>'`; opt a step out with `# allow-externalized-marker: <reason>` on the invoking `run:` or in the referenced script.
