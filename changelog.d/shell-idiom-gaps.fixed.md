- Shell-pattern hooks now catch equivalent banned idioms that previously slipped past: `check-stderr-suppression` flags the canonical `>/dev/null 2>&1`; `check-pinned-downloads` flags a bare `wget <url>` (saved to disk by default) and a `curl … > FILE` / `>> FILE` redirect; `check-workflow-pipefail` drops quoted-heredoc (`<<'EOF'`) bodies so a pipe outside is still seen and a `|` inside isn't a false positive, and only honors an `# allow-no-pipefail` opt-out in a real comment (not in a string or heredoc body); `check-unnamed-regex-groups` resolves `import re as x` / `from re import compile` aliases; `check-exit-suppression` catches `|| true` glued to a following metacharacter (`|| true&`).
