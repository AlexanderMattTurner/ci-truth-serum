- Several shell/YAML lints now catch cases they previously missed and no longer fire on legitimate code: `check-pipefail-grep-pipe` joins shell-continued lines, so a pipeline wrapped as `producer |` then `grep -q` on the next line is caught; `check-workflow-pipefail` requires `set -o pipefail` to run *before* the pipe it protects, so a `set -o pipefail` placed after the pipe (or in a never-reached tail) no longer falsely clears the step; `check-case-default` is now driven off the bash AST, so a single-line `case … esac` and compact same-line arms are checked; `check-pinned-downloads` recognizes a clustered short output flag (`wget -qO- URL | jq`), so writing to stdout is no longer mis-flagged as a disk save (while `-qO- URL > file` is still caught); `check-flag-arity` requires an arity guard to prove the *highest* positional the arm reads (a `[[ $# -ge 2 ]]` guard before a `$3`/`shift 3` read is now flagged) and no longer trusts a locally-defined `error()`/`fail()` helper as a bail when its body only warns and returns 0.
- `check-drift-guards`: the non-Python (JS/TS/shell) phrase pass is now scoped to test-shaped paths, so ordinary production prose (a `# kept in sync with deploy.yaml` comment) is no longer mistaken for a copies-agree test.
- README `.pre-commit-config.yaml` examples now use a `<latest release tag>` placeholder with a note pointing to the Releases page, instead of a `v0.1.0` tag that does not exist — so a copy-pasted config no longer fails on an unresolvable `rev`.
