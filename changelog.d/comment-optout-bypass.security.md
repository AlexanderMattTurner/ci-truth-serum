- `check-pinned-downloads` and `check-pipefail-grep-pipe` no longer accept an opt-out or verification token that lives in a comment or a bare substring, closing two supply-chain lint bypasses. `check-pinned-downloads` now classifies comment vs. code with the bash grammar (`tree-sitter-bash`) before scanning, so a verification token mentioned only in a comment (`# TODO: verify with sha256sum`) no longer satisfies the gate — the `sha256sum`/`cosign`/`gpg` check must sit in executed code. Both lints now route their `# pin-exempt:` / `# pipefail-grep-ok:` opt-outs through one shared comment-scoped matcher that requires the token inside a real `#` comment with a stated reason, so a token smuggled into a URL path or string (`curl -o x https://cdn/pin-exempt/x | sh`) can no longer silence the check.
