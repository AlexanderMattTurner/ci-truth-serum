- The shared `tree-sitter-bash` wrapper now folds every supplementary-plane (non-BMP, codepoint ≥ U+10000) character to U+FFFD before parsing. tree-sitter-bash 0.25's C scanner corrupts its heap on such a character next to a word-opening token (e.g. a `{` immediately followed by an astral char), segfaulting the whole process non-deterministically — so `check-case-default`, `check-workflow-pipefail`, and `check-flag-arity` could crash a pre-commit run (or a pytest worker) on hostile or generated input. Folding the char class out up front keeps every shell lint total; a supplementary-plane char is a plain word byte to bash, so the substitution never changes a lint's verdict.
