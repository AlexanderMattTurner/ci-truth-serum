- Hooks that hand-parse workflow YAML now close several fail-opens: `check-always-reporter` and `check-required-reporter` also check list-form `on: [pull_request, …]` (previously only mapping-form `on:` was inspected, so a gated required-check workflow missing its `always()` reporter slipped through); `check-concurrency` now flags a **job-level** `concurrency:` block that omits `cancel-in-progress`, not just the workflow-level one; `check-claude-model` anchors each violation to its real step via the parsed source line instead of a positional grep that a commented-out `uses:` line could misalign; and all eight of these lints report a workflow they can't parse as YAML as a violation instead of crashing with a traceback. The `cancel-in-progress-not-required`, `static-concurrency-ok`, and `concurrency-not-required` opt-outs are now honored only on a real `#` comment line, not anywhere in the file's bytes.
